# لیگ اتحاد (Ettehad League)

نسخه‌ی **کاملاً مسطح (flat)** — تمام فایل‌های HTML، CSS، جاوااسکریپت، آیکون‌ها، Manifest و Service Worker مستقیماً در همین یک پوشه قرار دارند؛ هیچ زیرپوشه‌ای وجود ندارد. مخصوص آپلود راحت از گوشی به گیت‌هاب.

بک‌اند: **Supabase** (Postgres + Auth + Storage، رایگان). بدون نیاز به Node.js، npm یا هیچ مرحله‌ی Build ای — فقط فایل‌های استاتیک.

## فهرست فایل‌ها

| فایل | نقش |
|---|---|
| `index.html` | تنها فایل HTML اپ (SPA) |
| `style.css` | تم شیشه‌ای مشکی/طلایی/آبی، راست‌چین |
| `manifest.json` | تنظیمات PWA برای نصب روی گوشی |
| `service-worker.js` | کش فایل‌ها برای کارکرد آفلاین |
| `offline.html` | صفحه‌ی نمایش‌داده‌شده وقتی اینترنت قطع است |
| `icon-*.png` | آیکون‌های PWA |
| `supabase-config.js` | اتصال به Supabase (کلیدها را اینجا وارد کنید) |
| `supabase-schema.sql` | جدول‌ها، قوانین امنیتی (RLS) و تابع ثبت نتیجه — یک‌بار در Supabase اجرا می‌شود |
| `auth-state.js` | مدیریت ورود مدیر/بازیکن |
| `access-guard.js` | محافظت از صفحات مدیر/بازیکن |
| `firestore-service.js` | تمام عملیات خواندن/نوشتن (پشت‌صحنه با Supabase؛ نام فایل تاریخی است) |
| `standings.js` | محاسبه‌ی خودکار جدول لیگ و گلزنان از روی نتایج |
| `date-utils.js` | تبدیل تاریخ میلادی به شمسی |
| `router.js` | مسیریابی مبتنی بر هش |
| `ui-helpers.js` | توابع کمکی رندر |
| `component-*.js` | تکه‌کدهای قابل‌استفاده‌ی مجدد |
| `page-*.js` | هر فایل یک صفحه‌ی کامل از اپ |
| `nav-footer.js` | نوار بالا و پایین سایت |
| `app.js` | نقطه‌ی شروع؛ همه‌چیز را به هم وصل می‌کند |

## گام ۱: ساخت پروژه‌ی Supabase

۱. به [supabase.com](https://supabase.com) بروید، ثبت‌نام کنید و یک پروژه‌ی جدید بسازید (رایگان).
۲. از منوی سمت چپ **SQL Editor** را باز کنید، محتوای فایل `supabase-schema.sql` را کامل کپی/پیست کنید و **Run** بزنید.
   - این کار جدول‌های تیم/بازیکن/مسابقه/خبر، قوانین امنیتی (RLS)، تابع اتمیک ثبت نتیجه، و باکت‌های Storage را می‌سازد.
۳. از **Authentication > Users > Add user** یک حساب Email/Password برای خودتان (مدیر لیگ) بسازید و **UID** آن را کپی کنید.
۴. در همان SQL Editor، این کوئری را با UID واقعی اجرا کنید تا مدیر مشخص شود:
   ```sql
   create or replace function public.is_admin()
   returns boolean language sql stable as $$
     select auth.uid() = 'همان-UID-مدیر'::uuid;
   $$;
   ```
۵. از **Project Settings > API**، مقادیر **Project URL** و **anon public key** را کپی کنید.

## گام ۲: تنظیم `supabase-config.js`

```js
export const SUPABASE_URL = "https://xxxx.supabase.co";
export const SUPABASE_ANON_KEY = "eyJhbGciOi...";
export const ADMIN_UID = "همان UID مدیر از گام ۱";
```

> کلید anon عمومی و غیرمحرمانه است — امنیت واقعی توسط قوانین RLS در `supabase-schema.sql` تأمین می‌شود، نه با پنهان‌کردن این مقادیر. پس مشکلی ندارد این فایل مستقیماً در گیت‌هاب باشد.

## گام ۳: آپلود به گیت‌هاب از گوشی

تمام فایل‌های این پوشه را (بدون هیچ زیرپوشه‌ای) در ریشه‌ی یک ریپازیتوری جدید آپلود و Commit کنید.

## گام ۴: انتشار (Hosting رایگان)

چون هیچ Build ای لازم نیست، هر سرویس استاتیکی کار می‌کند:
- **Vercel:** ریپازیتوری را Import کنید؛ چون فریمورکی تشخیص داده نمی‌شود، به‌عنوان استاتیک منتشر می‌شود.
- **GitHub Pages:** از تنظیمات ریپازیتوری، Pages را از شاخه‌ی اصلی فعال کنید.
- **Netlify:** ریپازیتوری را وصل کنید، پوشه‌ی publish را روی ریشه (`/`) بگذارید.

## گام ۵: افزودن حساب برای بازیکنان

چون از سمت کلاینت نمی‌توان برای شخص دیگری حساب ساخت:
۱. در Supabase از **Authentication > Users > Add user** یک حساب برای بازیکن بسازید و UID را کپی کنید.
۲. در **پنل مدیر > مدیریت بازیکنان**، پایین کارت همان بازیکن، UID را وارد کرده و «اتصال» را بزنید.

## گام ۶: نصب PWA روی گوشی

- **اندروید (Chrome):** منوی سه‌نقطه → «Add to Home screen»
- **آیفون (Safari):** دکمه‌ی Share → «Add to Home Screen»
- **کامپیوتر:** آیکون نصب کنار نوار آدرس

## اجرای محلی (اختیاری)

```bash
npx serve .
# یا
python3 -m http.server 8000
```

## نکات امنیتی
- اسم/ایمیل مدیر هیچ‌جای رابط کاربری نمایش داده نمی‌شود؛ فقط UID او در `supabase-config.js` و در تابع `is_admin()` قرار می‌گیرد.
- نوشتن‌های حساس (تیم/بازیکن/مسابقه/خبر) با Row Level Security محدود به همان UID است، حتی اگر رابط کاربری دور زده شود.
- ثبت نتیجه‌ی مسابقه از طریق یک تابع Postgres اتمیک (`submit_match_result`) انجام می‌شود تا آمار بازیکنان هیچ‌وقت ناهماهنگ نشود.
- بازیکنان فقط اجازه‌ی خواندن سند اتصال حساب خودشان را دارند.
